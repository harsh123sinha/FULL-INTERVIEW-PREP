Class selectors: Defined with a dot (`.`), such as `.example`. They can be applied to multiple elements on a page, allowing for shared styles across multiple elements.

ID selectors: Defined with a hash (`#`), such as `#example`. IDs should be unique to each element and provide a higher specificity, overriding class selectors when both are applied to the same element.

The CSS box model is a structure that represents the layout of HTML elements. It consists of:

  • Content: The area where text and images appear.
  • Padding: Space between content and the border, making the content appear larger.
  • Border: Surrounds padding and content, giving an outline to the element.
  • Margin: The outermost space that separates the element from other elements.

The `position` property determines how an element is positioned on a page. The values include:

  • static: Default; elements are positioned according to the normal document flow.
  • relative: Positioned relative to its normal position.
  • absolute: Positioned relative to its nearest positioned ancestor.
  • fixed: Positioned relative to the viewport, staying in place when scrolling.
  • sticky: Acts like relative until it reaches a specified scroll position, then becomes fixed.

Margin: The space outside the element’s border, creating separation between elements.

Padding: The space inside the element’s border, adding space between content and the border.

The `display` property defines how an element is displayed. Common values include:

  • block: Displays as a block-level element, taking up the full width.
  • inline: Displays inline with no line break and only takes up as much width as needed.
  • inline-block: Like `inline`, but allows setting width and height.
  • flex: Displays as a flex container for creating flexible layouts.
  • none: Hides the element.

Flexbox is a layout model in CSS that allows for creating flexible and responsive layouts. It is ideal for centering items, creating equal-width columns, and distributing space evenly in one-dimensional layouts. Common properties include `justify-content`, `align-items`, and `flex-direction`.

The `z-index` property controls the stacking order of positioned elements (with `position: relative`, `absolute`, or `fixed`). Higher `z-index` values appear above elements with lower values. It is useful when elements overlap and specific elements need to be visible on top.

Media queries allow for responsive design by applying different styles based on screen size, orientation, or resolution. They enable creating layouts that adjust for desktops, tablets, and mobile devices. Example: `@media (max-width: 600px) { ... }`.

CSS specificity determines which styles are applied when multiple rules target the same element. It follows a hierarchy:

  • Inline styles (highest specificity).
  • ID selectors.
  • Class selectors and pseudo-classes.
  • Element and pseudo-element selectors (lowest specificity).
The rule with the highest specificity is applied.

inline: Elements only take up as much width as their content, and do not start on a new line.

inline-block: Behaves like `inline` but allows setting width and height.

block: Takes up the full width of its container and starts on a new line.

CSS Grid is a two-dimensional layout system in CSS, ideal for complex layouts that require precise positioning both horizontally and vertically. It is useful for creating structured layouts and allows for more control than Flexbox over rows and columns.

The `overflow` property controls what happens when content overflows an element’s box. Common values are:

  • visible: Default; content is not clipped and may overflow.
  • hidden: Content is clipped and not visible beyond the box.
  • scroll: Adds a scrollbar to view overflowing content.
  • auto: Adds a scrollbar only if necessary.
